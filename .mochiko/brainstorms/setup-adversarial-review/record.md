- **Status:** accepted 2026-07-18 — acceptance ratified D7 as revised; landed: ROADMAP Key Decisions row + `BACKLOG.md` build items (build pending)

- **Topic:** whether and how `/mochiko:setup` gets an adversarial gap-finding review, analogous to the end-stage review in `/mochiko:brainstorm`.

## Context (opening read, from the files)

- `setup.md` (current, post-v3): the authoring loop already has an independent **validator seat** (`mochiko:validator` × `mochiko:validation-constitution`) — but its standing is **Tier-2 checklist grading**: trace closure both ways, region-marker integrity, index→file existence, ledger Three-Part completeness, anti-pattern/placeholder scans, semver. Mechanical fidelity, not gap-hunting.
- The **interrogation → synthesis** path has **no adversarial pressure at all**: the only check on `governance-intent.md` is G3 — the user confirming their own synthesis. The interrogation's card arbitration (recommend-then-arbitrate) is structurally the highest passive-acceptance-risk stretch in the pipeline.
- `brainstorm.md` (v2.2 form): review = cold reviewers at convergence (`mochiko:devils-advocate` × `mochiko:validation-brainstorm`), user-sized at a named gate (pair / single / none-with-waiver), one-shot cross-exam, survivors routed by answer-owner, dispositions + verify pass.

## Decisions

- **D1 — Target surface: the ratified synthesis, at G3.** The adversarial review stress-tests `governance-intent.md` before anything is authored — missed dimensions, unchallenged tier calls, passive card acceptances, too-easily-resolved reality conflicts. Not the authored surface set: a gap caught at G3 costs one interrogation follow-up; the same gap post-authoring costs re-ratification plus a produce→validate round, and setup's trace doctrine (selection belongs to the session, formulation to the producer) means artifact-side gap-hunting re-litigates a ratified contract from the wrong side. `Confident` *(adoption of lead recommendation, unelaborated — streak count 1)*
  - *Open thread — closed `Contested` at review (S4):* formulation quality (enforceable-in-practice vs. checklist-present) stays invisible to both G3 and the Tier-2 validator. The review showed the original dismissal conflated re-litigating *selection* with reviewing *formulation*, and verified the Tier-2 validator passes enforceability theater (checks mechanism *presence* only). User ruled: accept the risk, all tiers. Not to be re-raised.

- **D2 — Substrate: the synthesis, enriched with confidence marks.** Each GI element carries a mark from brainstorm's vocabulary (`Confident / Assumed / Contested / Unsure / Deferred`), assigned by the lead at assembly — `analysis-iterative` already obligates forming these indicators; today they're discarded at synthesis time. Small template delta; amend updates marks delta-wise. **The thin-rationale hunt survives into the reviewer's brief** (marks are self-reported by the reviewed lead — a sloppy lead yields falsely-confident marks, so echo-rationales in ruling columns stay an independent signal). Rejected: as-is synthesis (can't distinguish reasoned from waved-past rulings) and an as-you-go interrogation record (duplicates the synthesis's job; the synthesis *is* the session's record). `Confident` *(adoption of lead recommendation, unelaborated — streak count 2)*

- **D3 — Sizing: a tier-driven sizing gate.** Brainstorm's gate and menu (pair / single / none-with-recorded-waiver) imported, but the default recommendation is keyed to the declared tier (GI-001): `poc`/`internal` → single, `production`/`regulated` → pair. The user still rules at the gate. Rationale: setup's own thesis applied to itself — the declared risk posture drives how hard the intent gets stress-tested, giving the sizing recommendation an auditable basis. Rejected: as-is import (leaves the tier↔review link implicit), fixed single with no gate (forecloses the recorded-waiver discipline; a `poc` throwaway may honestly not warrant review). `Confident` *(adoption of lead recommendation, unelaborated — streak count 3 → streak flagged to user; next fork posed without a marked recommendation)*

- **D4 — Procedure home: a new dedicated skill.** The hunt procedure gets its own skill rather than a spawn-brief inline (violates procedure-into-skills + command-altitude) or an extension of `validation-brainstorm` (the shared part — adversarial persona + loop idiom — is already single-sourced in the `devils-advocate` agent and `loop-discipline`; the unshared parts fork on every section: substrate, hunt classes, fitness check, reality grounding, lens split). Precedent *(corrected at review, S11 — the original citation, "the grader family shares one agent (`validator`) while each artifact type gets its own `validation-*` skill," was factually wrong: `validator` runs only validation-constitution)*: the library's established pattern is **one reviewer agent hosting one skill per review target** — `devils-advocate` already hosts five; a sixth per-target skill follows it. The decision stands on this corrected ground plus trigger-reliability (two crisp per-artifact descriptions fire more reliably than one dual-trigger merge). Ruled after a B-vs-C discriminating exchange — streak broken by genuine engagement; the user additionally initiated D5's categorization requirement. `Confident`

- **D5 — (user-initiated; membership re-derived at review — S1 Critical, user re-ruled on corrected facts) The library's review-shaped skills split into two named families: `validation-*` vs `review-*`.**
  *The originally ratified membership table was built on a factually false premise the lead supplied (see Review S1: three of the four "stay" skills are devils-advocate-run, 3-state, lead-adjudicated — not validator/binary) and is superseded by the scheme below, which the user accepted at survivor routing.*
  - **Boundary** (the classification the names encode): ***who owns the clearing.*** `validation-*` = the skill's own verdict clears — binary checklist grade, default FAIL, persona `mochiko:validator`. `review-*` = severity-ranked findings / recommended status that a human or the lead adjudicates — the verdict never clears anything by itself (personas `mochiko:devils-advocate`, plus `mochiko:principal-architect` for feasibility).
  - **Prefix:** `review-*` (over `advocacy-*` / `stress-*` / `critique-*` — plain verb, matches the library's verb-prefix idiom).
  - **Membership (corrected):** `validation-constitution` **stays** — today's sole grader. Into `review-*`: **`review-specifications`** (was analysis-specifications) · **`review-plan-artifacts`**, **`review-task-artifacts`**, **`review-slices`** (were validation-*) · **`review-brainstorm`** (was validation-brainstorm) · **`review-feasibility`** (was validation-feasibility — 3-state, explicitly "input, not the gate" per plan.md) · **`review-governance-intent`** (new, D4).
  - **Build scope (S5):** the lockstep rename set is the full live-ref surface — each skill's directory + `name:` field · `agents/devils-advocate.md` `skills:` frontmatter and body refs · the router (`skills/mochiko/SKILL.md`) · command briefs (`specify.md`, `plan.md`, `tasks.md`, `slice.md`, `brainstorm.md`) · sibling-skill cross-refs including two `description`-field disambiguators (`analysis-iterative/SKILL.md`, the slices skill) that are model-invocation trigger text and dangle silently · REGISTRY rows. Also aligns `specify.md`'s done-condition wording with the boundary (clearing = the human gate + lead adjudication, not the reviewer's `ready` by itself). **(S6)** Dated narrative inside ROADMAP / BACKLOG / REGISTRY change-logs stays period-accurate, untouched.
  - Fixes a pre-existing inconsistency that turned out broader than first stated: five adversarial reviewers wore the grader prefix or a third prefix; after the split, prefix = family for every member. `Confident` — re-ratified post-review on corrected facts.

- **D6 — Amend runs: event-scaled sizing.** The gate always opens on amend; the recommendation is keyed to tier × delta weight — a governance event (tier bump, un-waive, floor/waiver change) gets D3's full tier-keyed default; lighter deltas recommend single at `production`/`regulated`, none-with-waiver below. Composes D3 (declared risk drives review strength) with amend doctrine (scope follows the delta) and reuses the governance-event boundary `setup.md` already draws. Rejected: uniform (predictably over-reviews light deltas, training the user to wave the gate off) and events-only — *primary ground corrected at review (S12):* the always-open gate keeps review-absence **audit-complete** (every amend records a sizing ruling or waiver); "removes user elevation" was refutable and is demoted to secondary. **(S7)** On a tier-bump amend, the revisit discipline extends beyond waivers: tier-loosened deck rulings are re-dealt (or force-re-marked) on the bump's agenda slice — a stale `Confident` mark must not carry a poc-era loosening into production unexamined. `Confident` *(adoption of lead recommendation, unelaborated — new streak count 1)*

- **D7 — (revised per review S2/S3/S8/S10/S13; ratified whole at user acceptance, 2026-07-18) The assembled flow.** `Confident`
  1. Assemble the synthesis with a confidence mark per GI element (D2 template delta).
  2. Sizing gate: lead states the synthesis's weight — *purpose (S10): informing the user's elevation ruling, not the default* — and recommends per tier (D3) / tier × delta on amend (D6); user rules pair / single / none → `none` = recorded waiver, proceed to G3.
  3. Cold review: `devils-advocate` seat(s) per ruling, briefed to run `review-governance-intent` against the frozen synthesis + interrogation agenda + `codebase-analysis.md` (brownfield). Synthesis frozen from spawn until dispositions land.
  4. Survivor routing by answer-owner: user rulings (deck/tier/waiver) → user directly; **challenged dimension prunes / scope decisions → user directly (S8)**; lead formulation → lead argues, two-exchange cap, tie-break; fact disputes → *reality-surface* facts checked against the analysis or an Explore subagent, never argued — **user-declared facts (team size, risk posture, lifespan) → the user, as confirmation (S13)**. Overruled survivor → element marked `Contested`.
  5. Verify pass confirms folds landed → G3. **Owner named at build when the lens split is fixed (S13):** brainstorm keys the verify pass to its record-integrity lens, which setup's split (likely coverage vs. coherence) won't have — the rule cannot copy verbatim.
  6. **G3-edit rule (S3):** material content added or changed via G3-`edit` after the review gets a bounded delta-pass by the still-seated reviewer before confirmation; G3-`reject` re-enters the full flow (fresh sizing + review) by construction. "Ratifying a stress-tested synthesis" thereby stays true of everything ratified.
  7. `governance-intent.md` gains a **Review section** — sizing ruling, tally, survivor dispositions, or the waiver. Rationale: D3's recorded-waiver discipline needs a durable home; an amend run should see how hard the intent it amends was tested. Mirrors brainstorm's record Review section. (Recovery keys off this section's state.)
  8. **setup.md propagation (S2) — the edits D7 entails, named:** the flow gains the sizing-gate + review stage before G3; Contract **done-condition** adds "the sized review ran (or its waiver is recorded) and every survivor carries a disposition"; **Bounds** add the review caps (one cold read per reviewer · one cross-exam, four messages · two-exchange lead↔reviewer cap per survivor · one verify pass); **Human-gates** add the sizing gate + survivor rulings; **Recovery** gains rows keyed to the synthesis Review section's state (sized-not-run → respawn reviewers · folded-not-verified → verify pass · waived → G3).
  - Downstream genuinely unchanged: authoring loop, Tier-2 validator, G4, G5. Delegated to build time: the pair's lens split + exact brief fields; **(S9)** `review-governance-intent` omits brainstorm's "never raise a `Contested` decision" skip rule, or requires a Contested-rationale audit before honoring it; **(S11)** the pair cross-exam protocol is single-sourced between `review-brainstorm` and `review-governance-intent`.
